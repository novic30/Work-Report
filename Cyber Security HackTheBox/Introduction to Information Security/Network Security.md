# Network Security

## Definition
Network Security protects networks, devices, and data from unauthorized access, attacks, and disruptions.

### Goals
- Confidentiality
- Integrity
- Availability (CIA Triad)

---

# Key Network Security Components

| Component | Purpose |
|------------|----------|
| Firewall | Filters network traffic based on rules |
| IDS | Detects suspicious activity |
| IPS | Detects and blocks suspicious activity |
| VPN | Creates encrypted connections over public networks |
| Access Control | Restricts access to authorized users |
| Encryption | Protects data confidentiality |

---

# Important Technologies

## Firewall
- First line of defense
- Controls inbound and outbound traffic
- Cannot stop every attack

### Security+ Tip
Firewall = Traffic filter

---

## IDS (Intrusion Detection System)
- Monitors traffic
- Detects threats
- Generates alerts

### Security+ Tip
IDS = Detect only

---

## IPS (Intrusion Prevention System)
- Monitors traffic
- Detects threats
- Blocks malicious activity

### Security+ Tip
IPS = Detect + Prevent

---

## VPN (Virtual Private Network)
- Encrypts network traffic
- Protects data in transit
- Common for remote workers

### Security+ Tip
VPN = Secure remote access

---

## Access Control
Ensures only authorized users access resources.

### Includes
- Authentication (Who are you?)
- Authorization (What can you access?)

---

## Encryption
Protects sensitive data.

### Types
- Data in Transit
- Data at Rest

### Security+ Tip
Encryption supports Confidentiality.

---

# Security+ Exam Facts

- Firewall → Filters traffic
- IDS → Detects attacks
- IPS → Detects and blocks attacks

- Authorization → Determine permissions
- Encryption → Confidentiality

# Quick Memorization

## IDS vs IPS

IDS = Detect

IPS = Detect + Block

---

## AAA

Authentication → Who are you?

Authorization → What can you do?

Accounting → What did you do?

---

## CIA

Confidentiality → Encryption

Integrity → Hashing

Availability → Backups & Redundancy
